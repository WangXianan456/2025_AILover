### 现有脚本的功能说明：

1. **图片数据与资源管理**
   * `download_images.py`: 从爬取的角色数据或外部源批量下载角色的立绘和头像图片。
   * `optimize_images.py`: 对本地图片进行压缩优化（减少前端加载时间）。
   * `fix_image_ext_in_json.py`: 修复或统一 JSON 数据中图片字段的后缀名扩展名（例如 `.png`, `.webp` 等格式）。
   * `image_usage_report.py`: 生成本地图片的使用情况报告（排查哪些图片被遗落，哪些数据缺失配图）。

2. **角色数据校验与处理**
   * `export_characters.py`: 解析并在不同环境/格式间导出清洗好的角色 JSON 数据
   * `validate_characters.py`: 校验爬取下来的角色属性是否完整、数据结构是否符合后端要求。
   * `inspect_filter_fields.py`: 探索和筛选角色数据中可用于前端侧边栏分类/过滤的字段分布（像神之眼、命之座、体型等字段）。

3. **项目执行与启动**
   * `run-backend.ps1`: 已有的**开启后端服务器**脚本（通过 `uvicorn` 运行 `app.main:app`，跑在 `8000` 端口）。
   * `init-db.ps1`: 用于将爬取的静态角色数据 `characters.json` 或向量数据导入诸如 ChromaDB/SQLite 等数据库中，做项目初始化。

### 快速启动脚本
方便工作流程：

✅ **开启前端**： run-frontend.ps1
它会自动定位到 frontend 文件夹安装依赖并启动 Vite：
```powershell
# 运行方式：
./scripts/run-frontend.ps1
```

✅ **开启后端**： run-backend.ps1
它会自动进入 backend 安装 Python 依赖并在 8000 端口启动：
```powershell
# 运行方式：
./scripts/run-backend.ps1
```

✅ **开启爬虫**： run-spider.ps1
自动进入 genshin_spider 爬虫工作目录并执行 `scrapy` 开始抓取原神数据：
```powershell
# 运行方式：
./scripts/run-spider.ps1
```

直接在终端执行对应的脚本命令（`.\scripts\run-xxx.ps1`）就能够一键拉起相关的服务或任务了！
